# Frontend Best Practices

## Typography

- Do not use oversized headings that dominate the viewport unless explicitly requested.
- Default to balanced headline sizing that supports the layout instead of overwhelming it.
- Treat large display text as an exception, not the default.
- On desktop, avoid hero and section titles that visually occupy roughly half the screen height or width.
- Prefer strong spacing, hierarchy, and contrast over extreme font size.

## Visual Direction

- Keep the UI polished and intentional, but avoid dramatic type scales that make the interface feel heavy or awkward.
- Favor readability and proportion over visual shock.
- If a bold visual direction is used, keep it controlled and structurally useful.

## Project Preference

- For this repository, prefer moderate heading scales and tighter composition across landing, profile, and dashboard screens.
- If there is any doubt, choose the smaller and cleaner option.
